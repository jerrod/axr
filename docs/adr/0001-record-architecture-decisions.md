# 1. Record Architecture Decisions

**Status:** Accepted

## Context

The axr plugin makes architectural decisions that affect scoring behavior, plugin structure, and downstream consumers. These decisions need to be documented so future contributors understand why things are the way they are.

## Decision

We will use Architecture Decision Records (ADRs) in `docs/adr/` to document significant decisions.

## Consequences

- Decisions are discoverable and searchable
- New contributors can understand rationale without archeology
- The axr rubric itself scores repos on having ADRs (docs.decision-log)
